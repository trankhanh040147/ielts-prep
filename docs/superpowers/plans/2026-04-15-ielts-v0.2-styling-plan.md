### Task 3 — Style ModePicker + keyboard/touch fix

File: `apps/web/src/components/ModePicker.tsx`

- [ ] Replace `hoveredMode` state with two states:
  ```ts
  const [hoveredMode, setHoveredMode] = useState<PracticeMode | null>(null)
  const [pinnedMode, setPinnedMode] = useState<PracticeMode | null>(null)
  ```
  Info panel shows when `hoveredMode === m || pinnedMode === m`.

- [ ] Add `?` icon button inside each mode button (or adjacent):
  - `onClick`: `setPinnedMode(pinnedMode === m ? null : m)` (toggle)
  - `aria-label="More info about [mode name]"`
  - `aria-expanded={pinnedMode === m}`
  - Style: `ml-1 text-xs opacity-60 hover:opacity-100` (subtle)

- [ ] Mode buttons: pill style per spec. Use `aria-pressed` (already present) for active state styling.

- [ ] Info panel: apply `bg-indigo-50 border border-indigo-100 rounded-lg p-4` styles per spec.

- [ ] Verify: info panel toggles on click/tap; also shows on hover; keyboard focus on `?` button shows panel.
