# IELTS Prep v0.2 — Styling & UX Foundation Design Spec

**Date:** 2026-04-15
**Status:** Ready for implementation
**Owner:** Antigravity (UI execution)

---

## Context

v0.1 shipped a fully functional writing feedback loop. All components work and are tested — but are completely unstyled (bare HTML elements, no layout). v0.2 Task 1 applies a design system to make the app usable and polished.

**No new features. No backend changes.**

---

## Goals

- Apply consistent visual design across all 7 components
- Mobile-first responsive layout (single column → two column on desktop)
- Fix ModePicker info panel: make it accessible on keyboard and touch (currently hover-only)

## Non-Goals

- Word count indicator, diff view, band score, expanded prompt bank — all deferred to later v0.2 tasks
- Custom fonts or icon libraries
- Backend or API changes

## Design System

### Tech Choice

**Tailwind CSS v4** via `@tailwindcss/vite` plugin — zero-config with Vite, utility classes in JSX.

Install in `apps/web`:
```bash
npm install -D tailwindcss @tailwindcss/vite
```

`vite.config.ts` — add plugin:
```ts
import tailwindcss from '@tailwindcss/vite'
// plugins: [react(), tailwindcss()]
```

`src/index.css`:
```css
@import "tailwindcss";
```

`src/main.tsx` — add import:
```ts
import './index.css'
```

### Color Palette

| Role | Value |
|---|---|
| Page background | `bg-slate-50` |
| Card / panel background | `bg-white` |
| Primary text | `text-slate-900` |
| Secondary / meta text | `text-slate-600` |
| Border | `border-slate-200` |
| Accent (active, buttons, focus) | `indigo-600` |
| Strength indicators | `emerald-600` / `emerald-800` |
| Issue indicators | `amber-700` / `amber-800` |
| Error | `red-600` / `red-50` |

### Typography

System font stack — no custom fonts needed.

- Base body: `text-sm` or `text-base text-slate-900`
- Section headings: `text-base font-semibold text-slate-900`
- Labels / badges: `text-xs font-semibold uppercase tracking-wide`

### Spacing & Shape

- Card padding: `p-4` or `p-6`
- Gap between sections: `gap-4` or `gap-6`
- Rounding: `rounded-lg` (cards/panels), `rounded-md` (buttons/inputs), `rounded-full` (mode pills, badges)
- Shadows: `shadow-sm` on cards

## Component Specs

### ModePicker

**Mode buttons — pill style:**
```
flex gap-2 flex-wrap mb-4
```

Each button:
- Base: `inline-flex items-center gap-1.5 px-4 py-2 rounded-full text-sm font-medium border transition-colors`
- Inactive: `bg-white border-slate-200 text-slate-700 hover:border-indigo-300 hover:text-indigo-700`
- Active (`aria-pressed="true"`): `bg-indigo-600 text-white border-indigo-600`

**Info panel trigger — keyboard/touch fix:**

The current hover-only behavior breaks on mobile and keyboard navigation. Replace with:

1. Add a `?` icon button (`aria-label="More info"`) next to each mode label, rendered inline in the button or as a separate small button.
2. Clicking the `?` button (or pressing Enter/Space on it) **toggles** the info panel for that mode.
3. Hovering the mode button **still shows** the info panel on desktop (keep `onMouseEnter` / `onMouseLeave`).
4. Keyboard focus on the `?` button shows the panel; blur hides it (with a short delay so it's readable).

State: `const [activeInfo, setActiveInfo] = useState<PracticeMode | null>(null)` replaces `hoveredMode`. Panel shows if `activeInfo === m || hoveredMode === m`.

**Info panel:**
```
mt-3 p-4 rounded-lg bg-indigo-50 border border-indigo-100 text-sm
```
- Description: `text-slate-700 mb-2`
- Criteria list: `list-disc pl-4 text-slate-600 space-y-1 mb-2`
- Example blockquote: `italic text-slate-500 border-l-2 border-indigo-300 pl-3 mt-2`

### DraftEditor

Wrap textarea in a `<div>` with a visible label:

```jsx

  <label htmlFor="draft" className="block text-sm font-medium text-slate-700 mb-1">
    Your draft
  </label>

    id="draft"
    aria-label="Writing draft"
    value={draft}
    onChange={(e) => onChange(e.target.value)}
    rows={12}
```

### FeedbackPanel

```jsx
<div>
  <h2 className="text-base font-semibold text-slate-900 mb-3">Feedback</h2>
  {feedback.map((unit, i) => (
    <div key={i} className="rounded-lg border border-slate-200 bg-white p-4 shadow-sm mb-3 last:mb-0">
      <p className="font-medium text-slate-800 mb-3">{unit.targetText}</p>

      {unit.strengths.length > 0 && (
        <>
          <p className="text-xs font-semibold text-emerald-700 uppercase tracking-wide mb-1">Strengths</p>
          <ul className="list-disc pl-4 text-sm text-emerald-800 space-y-0.5 mb-3">
            {unit.strengths.map((s, j) => <li key={j}>{s}</li>)}
          </ul>
        </>
      )}

      {unit.issues.length > 0 && (
        <>
          <p className="text-xs font-semibold text-amber-700 uppercase tracking-wide mb-1">Issues</p>
          <ul className="list-disc pl-4 text-sm text-amber-800 space-y-0.5 mb-3">
            {unit.issues.map((issue, j) => <li key={j}>{issue}</li>)}
          </ul>
        </>
      )}

      <p className="text-sm italic text-slate-600 border-l-2 border-indigo-300 pl-3">
        {unit.revisionHint}
      </p>
    </div>
  ))}
</div>
```

## Accessibility Checklist

- All buttons have clear text labels (existing `aria-label="Writing draft"` kept)
- Focus rings visible (`focus:ring-2 focus:ring-indigo-500`)
- Error messages use `role="alert"`
- Mode info panel accessible via keyboard (? button) and touch (click-toggle)
- Color is not the only differentiator (labels used alongside colors for strengths/issues)
