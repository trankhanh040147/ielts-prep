## Data Model

### FeedbackUnit (`apps/api/src/types.ts`)

```typescript
// Before
type FeedbackUnit = {
  level: FeedbackLevel
  targetText: string
  strengths: string[]
  issues: string[]
  revisionHint: string
}

// After
type FeedbackUnit = {
  level: FeedbackLevel
  targetText: string        // sentence-level: exactly one sentence, copied verbatim
  strengths: string[]
  issues: string[]
  revision: {
    explanation: string     // why it's weak + the grammatical/rhetorical principle to fix it
    rewrites: string[]      // 2тАУ3 concrete alternative sentences
  }
}
```

## Backend Changes

### `geminiClient.ts` тАФ Prompt update

Key additions to the system/user prompt:

- For sentence-level requests: instruct Gemini to return **one `FeedbackUnit` per sentence** in the draft. `targetText` must be the **exact sentence copied verbatim** тАФ not a summary, not the whole paragraph.
- For each unit, return `revision` as `{ explanation, rewrites }` тАФ not `revisionHint`.
- `rewrites` should contain 2тАУ3 alternative phrasings that fix the identified issue.

### `feedbackMapper.ts` тАФ Mapper update

- Map `revision: { explanation, rewrites }` instead of `revisionHint`.
- Fallback on parse failure: one unit with empty `strengths`, empty `issues`, and `revision: { explanation: "Could not parse feedback.", rewrites: [] }`.

## Frontend Changes

### B1 тАФ `?` tooltip fix (`ModePicker.tsx`)

- Remove `pinnedMode` state entirely.
- Info panel shows only when `hoveredMode === m` (hover only).
- Style the info panel as an absolute-positioned tooltip rather than a layout-shifting block below the buttons (prevents jitter from layout reflow).

### B2 тАФ Hover affordance

- `apps/web/src/index.css`: add `button { cursor: pointer; }` globally.
- `HistoryList.tsx`: add `hover:bg-slate-50` to each history row.

### I1 тАФ Topic switcher (`PromptCard.tsx` + `App.tsx`)

New UI row inserted between ModePicker and the prompt card:

```
[ Prompt dropdown (flex: 1)          ] [ ЁЯФА Random ]
```

- Dropdown lists all prompts for the active mode (truncated to ~60 chars if long).
- "Random" button picks a random prompt from the bank, excluding the current one.
- Selecting a prompt or clicking Random: resets draft, feedback, and sessionId (same behaviour as mode switch).
- `App.tsx` holds a `promptIndex` state (number). `PromptCard` receives the resolved prompt string.
